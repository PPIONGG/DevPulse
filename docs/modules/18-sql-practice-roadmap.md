# 🚀 แผนพัฒนาโมดูล SQL Practice (Roadmap & Advanced Features)

เอกสารฉบับนี้รวบรวมแผนการอัปเกรดระบบ **SQL Practice** ในอนาคต เพื่อเพิ่มประสิทธิภาพการใช้งาน ความเป็นมืออาชีพ และยกระดับประสบการณ์การเรียนรู้ของผู้ใช้งาน (UX/UI)

---

## 1. ระบบ SQL Autocomplete (IntelliSense) 💡
เปลี่ยนจากระบบพิมพ์ข้อความธรรมดา ให้เป็น Editor ระดับมืออาชีพที่ช่วยแนะนำคำสั่งอัตโนมัติ

### 🎯 เป้าหมาย
- ลดความผิดพลาดในการสะกดชื่อตาราง/คอลัมน์ (Typo)
- ช่วยให้ผู้ใช้เขียน Query ได้รวดเร็วขึ้น (Developer Experience)
- ให้ความรู้สึกเหมือนใช้งาน VS Code หรือเครื่องมือจัดการ Database จริงๆ

### 🛠 เทคนิคการทำ (Technical Plan)
1. **Library:** เปลี่ยนจาก `textarea` ไปใช้ [CodeMirror 6](https://codemirror.net/6/)
2. **Schema-Awareness:**
   - ใช้ข้อมูลจาก **Metadata API** (ที่ดึงชื่อตาราง/คอลัมน์จาก Schema อัตโนมัติ) มาป้อนให้ Editor
   - เมื่อพิมพ์ชื่อตารางตามด้วยจุด (เช่น `employees.`) ระบบจะแสดงชื่อคอลัมน์ในตารางนั้นทันที
3. **Keyword Suggestions:** รองรับคำสั่ง SQL มาตรฐาน (SELECT, FROM, WHERE, JOIN, ฯลฯ)

---

## 2. การวิเคราะห์ประสิทธิภาพ (Performance & Explain Plan) 📊
ยกระดับผู้ใช้งานจาก "เขียนถูก" เป็น "เขียนดี"

### 🎯 เป้าหมาย
- สอนให้ผู้ใช้เข้าใจเรื่อง Query Optimization
- แสดงความแตกต่างระหว่างการเขียน Query ที่ได้ผลลัพธ์เหมือนกันแต่ใช้ทรัพยากรต่างกัน

### 🛠 ฟีเจอร์ที่แนะนำ
- **Performance Distribution Graph:** แสดงกราฟเปรียบเทียบความเร็วของ Query เรากับผู้ใช้คนอื่นๆ
- **Explain Plan Visualization:** แสดง "แผนการรัน" ของ PostgreSQL ในรูปแบบที่อ่านง่าย เพื่อดูว่ามีการใช้ Index หรือทำ Sequential Scan หรือไม่

---

## 3. ระบบช่วยจัด Format (SQL Formatter) 🧹
สร้างนิสัยการเขียนโค้ดที่สะอาดและอ่านง่าย

### 🎯 เป้าหมาย
- จัดรูปแบบ Query ที่ซับซ้อนให้สวยงามอัตโนมัติด้วยการคลิกเพียงครั้งเดียว

### 🛠 เครื่องมือที่แนะนำ
- ติดตั้ง `sql-formatter` ในฝั่ง Frontend
- เพิ่มปุ่ม **"Format SQL"** ในแถบเครื่องมือของ Editor

---

## 4. ระบบส่งออกข้อมูล (Data Exporting) 📥
ทำให้ระบบเป็นเครื่องมือที่ใช้งานจริงได้นอกเหนือจากการฝึกฝน

### 🎯 เป้าหมาย
- อนุญาตให้ผู้ใช้นำผลลัพธ์ที่ Query ได้ไปใช้งานต่อใน Excel หรือโปรแกรมอื่นๆ

### 🛠 ฟีเจอร์ที่แนะนำ
- **Export as CSV/JSON:** เพิ่มปุ่มดาวน์โหลดที่ตารางผลลัพธ์ (Result Table)

---

## 5. ความต่อเนื่องและการจูงใจ (Gamification & Flow) 🏆

### 🎯 เป้าหมาย
- กระตุ้นให้ผู้ใช้กลับมาฝึกฝนอย่างต่อเนื่อง (User Retention)

### 🛠 ฟีเจอร์ที่แนะนำ
- **Next Recommended Challenge:** ระบบแนะนำโจทย์ข้อถัดไปที่เหมาะสมกับระดับทักษะปัจจุบัน
- **Achievement Badges:** มอบเหรียญรางวัลเมื่อทำโจทย์ครบหมวดหมู่ (เช่น "Join Master", "Window Function Wizard")
- **Daily Streak Bonus:** มอบ XP พิเศษหากทำ Daily Challenge ต่อเนื่องกันหลายวัน
